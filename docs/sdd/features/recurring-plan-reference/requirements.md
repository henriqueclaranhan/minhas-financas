# Recurring Plan Reference Requirements

## User story

As a user, I want recurring planned expenses with the same description to remain distinguishable.

## Acceptance criteria

- [x] Recurring entries display a stable short reference derived from their source document ID.
- [x] The Planning desktop list, mobile cards, and breakdown use the same reference.
- [x] Recurring breakdown metadata also shows cadence day, category, and payment method.
- [x] One-time entries do not display a reference.
