# Database Schema & Data Models

This document outlines the main entities used across the application and how they map to our Firebase Firestore collections.

## Global Rules
- **Security:** All collections are restricted by Firestore Security Rules ensuring users can only read, write, and delete documents where `userId` matches their authenticated `uid`.
- **References:** Documents typically store foreign keys as strings (e.g., `userId`, `plannedExpenseId`).

## Collections

### 1. `transactions`
Represents individual financial movements (both income and expenses).

**Document Structure:**
```typescript
interface Transaction {
  id?: string;               // Automatically generated by Firestore
  userId: string;            // Reference to the authenticated user
  description: string;
  amount: number;            // Stored as float/number
  paymentMethod: string;     // e.g., 'Crédito', 'Débito', 'Pix'
  installments: number;      // 1 for single payment
  date: string;              // ISO format: YYYY-MM-DD
  type?: 'income' | 'expense';
  category?: string;         // E.g., 'food', 'transport', 'salary'
  plannedExpenseId?: string; // (Optional) Link to the planned expense that generated this
}
```

### 2. `plannedExpenses`
Represents future or recurring commitments that aren't realized transactions yet.

**Document Structure:**
```typescript
interface PlannedExpense {
  id?: string;
  userId: string;            // Reference to the authenticated user
  description: string;
  amount: number;
  dueDate: string;           // ISO format: YYYY-MM-DD
  isRecurring: boolean;
  recurrenceInterval: number;// in months (e.g., 1 for every month)
  status: 'pending' | 'paid';
  type?: 'income' | 'expense';
  category?: string;
  paymentMethod?: string;
  installments?: number;
}
```

### 3. `creditCards` (Planned/Implemented)
Represents the user's credit cards to track limits and invoices.

**Document Structure:**
```typescript
interface CreditCard {
  id?: string;
  userId: string;
  name: string;              // e.g., 'Nubank', 'Itaú'
  limit: number;             // Total card limit
  closingDay: number;        // Day of the month the invoice closes
  dueDay: number;            // Day of the month the invoice is due
}
```

## Data Flow Notes
- When a `PlannedExpense` is marked as 'paid', a new `Transaction` is generated in the `transactions` collection, and the `PlannedExpense` status is updated (or its next due date is calculated if it's recurring).
